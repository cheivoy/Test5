📢 通知頻道

可新增多個頻道，每個頻道貼上 Discord 的 Incoming Webhook 網址，並各自勾選要接收哪些通知、要不要 @ 身分組。「開放請假」通知會自動附上請假連結。

🔗 站台與伺服器

站台網址用來在通知附上正確的請假連結；伺服器 ID 用來讓機器人指令對應到你的戰隊（在 Discord 開啟開發者模式後，右鍵伺服器→複製伺服器 ID）。

綁多個伺服器就每行填一個；註冊指令時會對每個伺服器各註冊一次。
🤖 機器人查詢指令

在 Discord 綁定機器人後，成員可用 /查詢 名字、/出勤榜、/請假名單 直接查詢。指令在該伺服器的任何頻道都能用（不限一個頻道）。設定完成後按下方按鈕註冊指令（有填伺服器 ID 會即時生效；否則全域最多等 1 小時）。

後端需要設定的環境變數（管理員看）
在 Worker 設定三個變數（Discord 開發者後台 → 你的 Application）：
• DISCORD_APP_ID：Application ID
• DISCORD_PUBLIC_KEY：Public Key（給互動驗簽用）
• DISCORD_BOT_TOKEN：Bot Token（Secret）
並把 Interactions Endpoint URL 設為 你的Worker網址/api/discord/interactions。
